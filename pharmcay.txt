🧠 PROJECT OVERVIEW

I am building a multi-business, multi-branch pharmacy management system (SaaS-style ERP) using:

React (Frontend)
Node.js + Express (Backend)
MongoDB (Database)
React Query (Server state management)
Redux Toolkit (Global client state)
Axios (API calls)

The system includes:

Multi businesses (organizations)
Each business has multiple branches
Each branch has independent dashboard
Inventory management (15+ CRUD modules)
Sales, orders, customers, stock, reports
Business + branch selector dropdown (global filter)
🎯 CORE PROBLEM THIS ARCHITECTURE SOLVES
Fast data loading across tabs
Multi-business switching without lag
Branch-based data isolation
Scalable dashboard system
Avoid unnecessary re-renders & API calls
Smooth UX (instant feel like SaaS apps)
🏗️ ARCHITECTURE DESIGN RULES
🔴 RULE 1: STATE SEPARATION
✅ Redux (Global UI State ONLY)

Redux sirf ye cheeze store karega:

selectedBusinessId
selectedBranchId

user role/permissions (optional)

❌ Redux me API data NEVER store karna

✅ React Query (SERVER STATE ONLY)

React Query will handle ALL API data:

Inventory data
Dashboard stats
Sales data
Orders
Customers
Reports
Stock movements

👉 Rule:

“Everything that comes from backend = React Query”

🔁 GLOBAL FLOW (VERY IMPORTANT)
STEP 1: User selects Business + Branch

From dropdown:

dispatch(setBusinessId(id))
dispatch(setBranchId(id))
STEP 2: Redux updates global state
selectedBusinessId = "b1"
selectedBranchId = "br2"
STEP 3: React Query auto triggers API
useQuery({
  queryKey: ['inventory', businessId, branchId],
  queryFn: () => fetchInventory(businessId, branchId)
})
STEP 4: UI updates instantly
Data re-renders automatically
No full page reload
Smooth SaaS experience
⚡ PERFORMANCE OPTIMIZATION RULES
🔥 1. CACHING (VERY IMPORTANT)
staleTime: 5 * 60 * 1000 // 5 min cache

👉 Benefit:

Same data repeat fetch nahi hoga
Tabs instantly open honge
🔥 2. KEEP PREVIOUS DATA
keepPreviousData: true

👉 Benefit:

Branch change pe UI blank nahi hoga
Old data visible rahega while new loads
🔥 3. QUERY KEYS DESIGN
['inventory', businessId, branchId]
['dashboard', businessId, branchId]
['orders', businessId, branchId]

👉 Benefit:

Proper caching
No data mix
Accurate refetch
🔥 4. PREFETCHING (PRO LEVEL UX)
queryClient.prefetchQuery({
  queryKey: ['inventory', branchId],
  queryFn: fetchInventory
})

👉 Benefit:

User click se pehle data ready
Instant loading feel ⚡
🔥 5. LAZY LOADING / PAGINATION
1000 records ek sath mat lao
Table pagination use karo
Infinite scroll optional
🧩 API DESIGN RULES

Backend APIs should be:

GET /inventory?businessId=1&branchId=2
GET /dashboard?branchId=2
POST /inventory
PUT /inventory/:id
DELETE /inventory/:id

👉 Always filter by branchId + businessId
